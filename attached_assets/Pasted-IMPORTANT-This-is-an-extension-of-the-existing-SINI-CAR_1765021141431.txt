IMPORTANT — This is an extension of the existing SINI CAR system.
Do NOT delete any existing customer logic. We are ADDING a full Supplier Portal.

GOAL:
Implement a complete Supplier Portal experience with:
1) Supplier login + dashboard
2) Supplier-specific home page (Command 13 integration)
3) Add products (manual + Excel upload)
4) Update products and pricing
5) See purchase requests sent to this supplier
6) Respond to requests (approve, quote, reject)
7) Notifications + activity log integration (Commands 10 & 11)
8) Reports for suppliers (Command 9 compatible)
9) Supplier settings (Command 12 integration)

────────────────────────────────
1) Backend – Supplier Model Extension
────────────────────────────────

Extend User model to support supplier-specific fields:
- isSupplier (boolean)
- supplierType (LOCAL / INTERNATIONAL)
- supplierCompanyName
- supplierVat
- supplierCrNumber
- supplierCatalogUrl (optional)
- supplierRating (numeric, default 0)
- supplierActiveProductsCount
- supplierTotalQuotesSubmitted

Do NOT remove any existing customer fields (suppliers may also be customers).

────────────────────────────────
2) Backend – Supplier Product Model
────────────────────────────────

Create new table: SupplierProduct
Fields:
- id
- supplierId (FK to User)
- sku (string)
- oemNumber (string)
- name
- category
- brand
- model
- yearFrom
- yearTo
- purchasePrice (price supplier sells to you)
- minOrderQty
- stock
- deliveryTime (days)
- isActive (boolean)
- createdAt
- updatedAt

NOTES:
- DO NOT replace main SINI CAR products.
- Supplier products are a separate layer that merge later when needed.

────────────────────────────────
3) Backend – Supplier Product API
────────────────────────────────

A) POST /api/supplier/products
   Allows supplier to add a new product (manual form).

B) POST /api/supplier/products/import
   - Upload Excel file with many products.
   - Parse columns:
     • sku
     • oemNumber
     • name
     • price
     • stock
     • category
     • brand
     • model
     • yearFrom/yearTo
   - Validate and insert/update.

C) GET /api/supplier/products
   - With filters:
     • search
     • category
     • brand
     • availability
   - Pagination required.

D) PUT /api/supplier/products/:id
   - Update price, stock, details.

E) DELETE /api/supplier/products/:id
   - Soft delete → isActive = false

────────────────────────────────
4) Backend – Supplier Requests Module
────────────────────────────────

Create new table: SupplierRequest
Fields:
- id
- requestId (FK to main PurchaseRequest)
- supplierId
- status (NEW / VIEWED / QUOTED / REJECTED)
- quotedPrice
- stockAvailable
- notes
- createdAt
- updatedAt

Logic:
- When a customer submits a purchase request:
  • System selects which suppliers are eligible (based on brand/model/type).
  • Create a SupplierRequest for each selected supplier.

Supplier actions:
- View assigned requests:
  GET /api/supplier/requests

- Submit quote:
  PUT /api/supplier/requests/:id/quote
  Fields:
    • quotedPrice
    • stockAvailable
    • notes

- Reject:
  PUT /api/supplier/requests/:id/reject

Triggers:
- Notifications to admin (Command 11)
- Logging into Activity Log (Command 10)
